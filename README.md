**Your AI agent wants to overwrite a file. ClawGuard stops it, shows you exactly what changes, and waits for your go-ahead.**

---

## The Problem

AI agents that edit files are powerful — and dangerous. Without oversight:

- **Files get overwritten** without you seeing the diff
- **Costs spiral** with no per-session or daily cap
- **Mistakes are applied immediately** with no undo

ClawGuard fixes all three.

---

## Features

- **Intercepts writes at the platform level** — Every `write`, `edit`, `apply_patch`, and `exec` call is captured via an OpenClaw `before_tool_call` hook before it reaches the filesystem
- **Staged diffs** — Changes are written to `/tmp/clawguard/staging/` and shown as unified diffs
- **Multi-channel approval** — Approve or reject via Telegram inline buttons, Discord components, or the web dashboard
- **Budget enforcement** — Per-model token cost tracking with daily limits, alert thresholds, and hard stops
- **SQLite-backed queue** — Pending changes survive restarts; history is preserved indefinitely
- **Real-time dashboard** — Live updates via SSE; dark-themed, responsive web UI
- **Zero JS framework** — Web UI uses vanilla JS + htmx; no React, no bundler required

---

## Architecture

ClawGuard runs as an **OpenClaw plugin**. It hooks into the agent runtime using the official `before_tool_call` hook — no proxy, no wrapper, no prompt tricks.

```
OpenClaw Agent
      │
      │ tool call: write("src/app.ts", newContent)
      ▼
┌─────────────────────────────────────────────────────┐
│               OpenClaw Plugin Runtime                │
│                                                      │
│  api.registerHook("before_tool_call")                │
│    ├─ Budget check ──────────────── BudgetTracker   │
│    ├─ Stage change ─────────────── StagingEngine    │
│    │     └── /tmp/clawguard/staging/{uuid}/         │
│    ├─ Generate diff ────────────── DiffGenerator    │
│    ├─ Queue entry ──────────────── SQLite DB        │
│    └─ Notify channels ──────────┬─ Telegram         │
│                                 ├─ Discord           │
│                                 └─ Web UI (SSE)      │
│                                                      │
│  await user decision...                              │
│    ├─ Approved → { block: false } → tool runs      │
│    └─ Rejected → { block: true  } → tool cancelled │
└─────────────────────────────────────────────────────┘
      │
      ▼
 Real filesystem (only if approved)
```

The approval web UI runs on `localhost:3847` as a background service started by the plugin itself.

## Quick Start

### 1. Clone and build

```bash
git clone https://github.com/your-org/clawguard
cd clawguard
npm install
npm run build
```

### 2. Configure ClawGuard

```bash
cp config.example.json5 config.json5
# Edit config.json5 — Telegram token, budget cap, etc.
```

### 3. Register as an OpenClaw plugin

Edit `~/.openclaw/openclaw.json` and add the following sections:

```json
{
  "plugins": {
    "load": {
      "paths": [
        "/path/to/clawguard/dist"
      ]
    },
    "entries": {
      "clawguard": {
        "enabled": true
      }
    }
  }
}
```

> **Important:** The `dist/` directory must contain `openclaw.plugin.json` (generated by `npm run build`) and `index.js` (the plugin entry point). The plugin entry is what OpenClaw loads — it is **not** the HTTP server starter.

### 4. (Optional) Add the skill

The skill gives agents an additional textual instruction to route changes through ClawGuard. It's a soft layer on top of the hard plugin interception — useful for agents that support skills.

In `~/.openclaw/openclaw.json`:

```json
{
  "skills": {
    "entries": {
      "clawguard": {
        "enabled": true,
        "env": {
          "CLAWGUARD_URL": "http://localhost:3847"
        }
      }
    }
  }
}
```

The skill file lives at `openclaw-skill/SKILL.md` in this repo. OpenClaw finds it through the `skills` path declared in `openclaw.plugin.json`.

### 5. Restart OpenClaw

```bash
openclaw gateway restart
```

Verify ClawGuard loaded without errors in the startup output. The approval dashboard will be accessible at `http://localhost:3847`.

## Configuration Reference

Copy `config.example.json5` to `config.json5`:

```json5
{
  channels: {
    telegram: {
      enabled: true,
      token: "YOUR_BOT_TOKEN",    // @BotFather
      chatId: "YOUR_CHAT_ID",
    },
    discord: {
      enabled: false,
      token: "YOUR_BOT_TOKEN",    // Discord Developer Portal
      channelId: "YOUR_CHANNEL_ID",
    },
    web: {
      enabled: true,
      port: 3847,
      host: "127.0.0.1",
    },
  },
  budget: {
    daily_limit_usd: 10.00,       // Hard cap per day
    alert_threshold_pct: 80,      // Alert at 80% of cap
    hard_stop: true,              // Block agent when cap reached
  },
  staging: {
    directory: "/tmp/clawguard/staging",
    auto_approve_read: true,      // Skip approval for read ops
    require_approval: ["write", "edit", "apply_patch", "exec"],
    timeout_seconds: 300,         // Auto-reject after 5 min (0 = never)
  },
  logging: {
    level: "info",                // "debug" | "info" | "warn" | "error"
    file: "~/.clawguard/clawguard.log",
  },
}
```

## How the Plugin Works Internally

OpenClaw loads `dist/index.js` from the plugin directory using [jiti](https://github.com/unjs/jiti). That file must export a default object:

```typescript
export default {
  id: "clawguard",
  configSchema: { /* JSON Schema */ },
  register(api) {
    // Start the approval web server
    api.registerService({ ... });

    // Hook into every tool call — runs before the tool executes
    api.registerHook(
      "before_tool_call",
      async (event, ctx) => {
        const { toolName, params } = event;

        // Only intercept write operations
        if (!requiresApproval(toolName)) return { block: false };

        // Stage the change, notify channels, wait for decision
        const approved = await stageAndAwaitApproval(toolName, params);

        return approved
          ? { block: false }
          : { block: true, blockReason: "Change rejected by ClawGuard." };
      },
      { name: "clawguard-before-tool-call" }
    );
  },
};
```

The hook return value `{ block: true }` is what actually prevents the tool from running — it is enforced by OpenClaw at the runtime level, not by any prompt instruction.

## Approval Channels

### Telegram

1. Create a bot with [@BotFather](https://t.me/BotFather) — copy the token
2. Get your chat ID (message [@userinfobot](https://t.me/userinfobot))
3. Set `channels.telegram.enabled: true` in `config.json5`

When a change arrives, you'll receive:

```
✏️ MODIFIED `src/core/staging.ts`
+12 / -3

```diff
@@ -45,7 +45,19 @@
   apply(changeId: string): void {
-    fs.copyFileSync(staged, real);
+    // Ensure parent directory exists
+    const dir = path.dirname(change.filePath);
+    if (!fs.existsSync(dir)) {
+      fs.mkdirSync(dir, { recursive: true });
+    }
+    fs.writeFileSync(change.filePath, change.newContent, 'utf-8');
```

[✅ Apply]  [❌ Reject]
```

**Commands:** `/budget` · `/pending` · `/history`

### Discord

1. Create an app at the [Discord Developer Portal](https://discord.com/developers/applications)
2. Add a bot, enable Message Content Intent
3. Invite the bot with `Send Messages` + `Use Slash Commands`
4. Set `channels.discord.enabled: true` in `config.json5`

### Web UI

Access at `http://localhost:3847` (or your configured port). No login required by default — bind to `127.0.0.1` to keep it local.

## Docker

```bash
# Build
docker build -f docker/Dockerfile -t clawguard .

# Run (mount config + data)
docker run -d \
  -p 3847:3847 \
  -v $(pwd)/config.json5:/app/config.json5:ro \
  -v clawguard-data:/root/.clawguard \
  --name clawguard \
  clawguard
```

> **Note:** When running in Docker, the plugin entry point (`dist/index.js`) is not loaded by OpenClaw — Docker mode runs the HTTP server standalone. The `before_tool_call` hook is only active when the plugin is loaded directly by the OpenClaw gateway process.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `plugin manifest not found: .../dist/openclaw.plugin.json` | `npm run build` wasn't run, or ran before the manifest was added to the build output | Run `npm run build` again; ensure `openclaw.plugin.json` is in `dist/` |
| `stale config entry ignored: clawguard` | `plugins.entries.clawguard` exists but `plugins.load.paths` doesn't point to a valid plugin dir | Add the correct path to `plugins.load.paths` or remove the stale entry |
| Agent writes files without going through ClawGuard | Plugin not loaded (check gateway startup logs) | Verify `plugins.entries.clawguard.enabled: true` and restart the gateway |
| Dashboard shows no pending changes | Plugin loaded but `before_tool_call` hook not firing | Check that the tool name matches the `require_approval` list in `config.json5` |
| `port 3847 already in use` | Multiple ClawGuard instances / stale process | `lsof -i :3847` and kill the stale process |
